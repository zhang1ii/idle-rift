# Idle Rift 试玩说明

当前版本用于验证“战前编排五技能程序 → 自动战斗 → 观察失败原因 → 调整装备与顺序”的核心循环，建议单次试玩 5～10 分钟。

## 启动

Windows 用户双击仓库根目录的 `PLAYTEST_WINDOWS.bat`。脚本会自动查找已安装的 Godot 4.7.x 并运行当前项目。

也可以手动执行：

```powershell
godot --path .
```

## 建议试玩路线

1. 在第 1 层调整六选五的技能组合和顺序，点击开始战斗，观察怒意、流血、回血与泄怒的循环。
2. 第 2 层观察腐血叠层带来的持续战损。
3. 第 3 层比较直接伤害与穿透厚甲的流血伤害。
4. 第 4 层观察敌方第三击前的重击预警，尝试让壁垒落在危险窗口附近。
5. 第 5 层挑战碎岩 Boss，观察技能格裂化是否打断完整回路。
6. 第 5 层首杀后按 `Tab` 查看天赋，按 `B` 查看、比较和更换装备。

战斗中不能更换装备、技能或使用药水。普通层战前可自动购买并使用药水；Boss 入场前会回满生命。死亡后挂机停止，需要返回战前重新调整。

## 反馈时请记录

- 当前楼层和角色装备平均阶级（G）。
- 五个技能的顺序，以及没有选择的第六个技能。
- 死亡或异常发生前，事件栏最后几条信息。
- 问题属于“数值打不过”“规则看不懂”“操作入口找不到”还是“画面表现异常”。

## 画面验收标准

- 我方始终锚定在画面左侧，攻击最多向前踏出 24 个逻辑像素，随后回到原位。
- 敌方始终锚定在画面右侧，刷新位移不超过 8 个逻辑像素。
- 同屏最多显示 4 个伤害数字。
- 返回战前或刷新敌人后，不应残留倾斜、受击颜色、护盾或命中特效。

## 当前已知限制

- 普通敌人暂时共用裂隙野猪美术，名称和机制已经区分，独立敌人资源仍待制作。
- 第 1～10 层是当前完整内容范围；第 11～15 层尚未制作。
- 存档、真实离线结算尚未接入，关闭游戏会丢失本次试玩进度。
- 为方便当前测试，界面可直接选择第 1～5 层；普通层“击杀 20 次后逐层解锁”的正式进度规则尚未接入完整界面。
