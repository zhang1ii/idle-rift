# AI Agent 工具链计划

第三方 Agent 项目作为开发辅助使用，不复制进游戏运行时代码，也不成为玩家运行游戏的依赖。

## 候选工具

### Godogen

参考其面向 Codex 的阶段式开发、运行、截图检查和修复流程。引入前先检查技能内容和脚本权限，只选取与 Godot 项目相关的部分。

### Godot AI

候选编辑器桥接层，用于读取场景树、操作节点、运行测试、捕获错误和截图。待本机安装 Godot 后再集成。

### Agent Sprite Forge

候选像素资产管线，用于角色、怪物、特效和 Sprite Sheet 原型。生成资产必须经过人工挑选、统一调色、透明边缘检查和帧动画验证。

## 使用边界

- 游戏逻辑和数据格式由本仓库掌控。
- 第三方工具升级不得直接改变存档格式。
- 自动生成内容必须能够追溯提示词、模型和后处理步骤。
- 所有外部代码和资产在进入项目之前检查许可证。
- 编辑器 Agent 只监听本机接口，不暴露到公开网络。

## 计划顺序

1. 完成纯 Godot 的最小战斗和掉落闭环。
2. 接入 Godot 编辑器 Agent，提高场景与测试迭代效率。
3. 建立可重复的像素资产生成和验证流程。
4. 在玩法稳定后再增加多 Agent 的设计、数值和 QA 流程。
