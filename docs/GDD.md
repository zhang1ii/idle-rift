# Idle Rift 最小玩法设计

## 1. 核心状态

玩家拥有一个英雄、一组已装备物品、一个背包和当前可进入的最高裂隙层数。

每层裂隙定义：

- 敌人等级与基础属性倍率
- 敌人组合
- Boss
- 掉落等级范围
- 通关条件

## 2. 自动战斗

英雄自动寻找目标、进行普通攻击并按优先级释放技能。玩家不需要实时操作。

首版战斗只需要验证：

- 攻击节奏是否清晰
- 技能是否容易辨认
- 装备提升是否能从击杀速度上被感知
- 失败原因是否能被理解

## 3. 掉落系统

### 装备部位

- 武器
- 头部
- 胸甲
- 戒指
- 护符

### 品质

- 普通：只有基础属性
- 魔法：1 至 2 条随机词缀
- 稀有：3 至 4 条随机词缀

传奇装备在核心循环验证后加入，避免首版同时承担随机系统和特殊机制的平衡成本。

### 首版词缀候选

- 最大生命
- 攻击伤害
- 攻击速度
- 暴击概率
- 暴击伤害
- 每秒生命恢复
- 击杀恢复生命
- 对 Boss 伤害
- 低生命时减伤
- 技能冷却缩减

## 4. 收菜与离线收益

离线结算基于离线前已验证的稳定击杀效率，而不是完整模拟每一场战斗。

结算内容：

- 离线时长
- 击杀数量
- 获得货币
- 装备数量与最高品质
- 因背包或筛选规则被自动分解的数量

首版设置离线收益时间上限，具体时长在测试后决定。

## 5. 推进规则

- 当前层稳定挂机，用于积累装备。
- 玩家主动发起下一层突破战。
- 击败 Boss 后解锁下一层。
- 失败后回到已解锁层继续挂机，不损失装备。

## 6. 首个开发里程碑

一个无正式美术的战斗沙盒：英雄自动攻击无限刷新的敌人，敌人死亡后生成一件装备数据，玩家可以查看并换装，换装会立刻改变击杀速度。
